#### 重构作用域和作用域方法 - 改为偏好使用成员方法作为作用域方法，而非带上下文参数的扩展方法

> - 版本：v2.1.8
> - 日期：2026-05-08

说明：
- 这里的作用域指的是某种接口或类，对应某种限定的API范围，某些情况下还带有自己的上下文。`SequenceScope` 和 `CoroutineScope` 都是典型的作用域。
- 区别于普通的工具方法和扩展方法，作用域方法只能在对应的作用域中使用。例如，将作用域对象作为 `this`，或者上下文参数 `context(scope: Scope)`。
- 改为偏好使用成员方法作为作用域方法，而非带上下文参数的扩展方法。除非不存在一个明确的可控制的作用域（类/接口）。

目的：
- 即使上下文中不存在符合条件的作用域对象，IDE 仍会补全作为扩展方法的那些作用域方法。
- 改为使用成员方法的话，可以避免这种非理想的行为，并且允许隐藏实现细节。

相关代码位置：
- `icu.windea.pls.config.select.CwtConfigSelectScope`
- `icu.windea.pls.lang.select.ParadoxSelectScope`

#### 重构 path/expression/complexExpression/config/configExpression - 更改解析器与解析方法的代码风格

> - 版本：v2.1.10
> - 日期：2026-06-14

说明：
- 将对外的解析方法改为模型接口上的静态方法，委托给解析器类（可能是单例对象，可能是私有的）的对应方法。
- 解析器类通常是同文件中的私有类，少数情况下也可以是不同文件中的公开类。
- 偏好上，将同文件中的私有实现方法放到公开接口方法之后，并且使用 `region...endregion` 注释包围。
- 偏好上，将文档注释放到接口代码上，实现代码（以及公开的解析器类的代码）通常不需要文档注释。

目的：
- 为了统一代码风格。

示例（不需要公开解析类的情况）：

```kotlin
interface Expression {
    val value: String
    
    companion object {
        @JvmStatic
        fun resolve(text: String): Expression = ExpressionResolver.resolve(text)
    }
}

// region Implementations

private object ExpressionResolver {
    fun resolve(text: String): Expression {
        return ExpressionImpl(text)
    }
}

private data class ExpressionImpl(override val value: String)

// endregion
```

#### 关于上下文对象（context object）

> - 版本：v2.1.10
> - 日期：2026-06-20

说明：
- 上下文对象应当被声明为数据类，其属性应尽可能不可变，可以有计算属性和一些工具方法。
- 上下文对象的切换是通过复制当前上下文，并按需更改特定的属性来达成的。
- 切换上下文时，不一定需要直接传递复制后的上下文对象，或是声明并传递不同名的局部变量。
- 一般来说，上下文的切换可以随编程语言（Kotlin）的作用域的切换而按需切换，直接声明同名的局部变量并覆盖（如在循环中）。

#### 集合选型（collection selection）

> - 版本：v3.0.1
> - 日期：2026-07-20

主要考量是在特定场景下的内存占用的细节，其次是作为一种统一的规范。

- `ArrayList` - 不需要保证线程安全的可变列表的默认选择。通常是但并不总是符合前述条件时的最佳选择。
- `SmartList` - 当长度为1时，相比 `ArrayList` 有着更少的内存占用。如果存在大量长度为1的不需要保证线程安全的不可变列表，这会是更好的选择。
- `ImmutableList` (guava) - 不可变列表，拥有更少的内存占用。对于反序列化后的索引数据、不可变缓存、以及部分轻量的无需缓存的计算结果，这会是更好（且优先）的选择。
- `IntArrayList` 等 (fastutil) - 不适合直接使用数组，且元素是原始类型时，这会是更好（且优先）的选择。
- `ObjectArrayList` (fastutil) - 相比 `ArrayList` 并没有明显的性能优势，仅在绝对必要时使用。例如需要使用到独有的API（如 `trim()`）。或者，作为一种统一的规范。
- `HashSet` - 不需要保证线程安全且不需要保证元素顺序的可变集的默认选择。通常是但同样并不总是符合前述条件时的最佳选择。
- `LinkedHashSet` - 在前者的基础上，如果需要保证元素顺序。
- `ImmutableList` (guava) - 不可变集，拥有更少的内存占用。对于反序列化后的索引数据、不可变缓存、以及部分轻量的无需缓存的计算结果，这会是更好（且优先）的选择。
- `IntOpenHashSet` 等 (fastutil) - 元素是原始类型时，这会是更好（且优先）的选择。
- `ObjectOpenHashSet` (fastutil) - 拥有相对较少的内存占用。如果存在大量集，可以考虑使用。
  - 参见：`com.intellij.util.containers.CollectionFactory.createSmallMemoryFootprintSet()`
- `ObjectOpenLinkedHashSet` (fastutil) - 在前者的基础上，如果需要保证元素顺序。
  - 参见：`com.intellij.util.containers.CollectionFactory.createSmallMemoryFootprintLinkedSet()`
- `ObjectLinkedOpenCustomHashSet` (fastutil) - 可以用来创建大小写不敏感的字符串集。
  - 参见：`com.intellij.util.containers.CollectionFactory.createCaseInsensitiveStringSet()`
- `ObjectArraySet` (fastutil) - 拥有更少的内存占用。如果存在大量集，并且重排序时的性能相比之下并不重要，这会是更好的选择。
- `ImmutableMap` (guava) - 不可变映射，拥有更少的内存占用。对于反序列化后的索引数据、不可变缓存、以及部分轻量的无需缓存的计算结果，这会是更好（且优先）的选择。
- `Int2ObjectOpenHashMap` 等 (fastutil) - 元素是原始类型时，这会是更好（且优先）的选择。
- `Object2ObjectOpenHashMap` (fastutil) - 拥有相对较少的内存占用。如果存在大量映射，可以考虑使用。
  - 参见：`com.intellij.util.containers.CollectionFactory.createSmallMemoryFootprintMap()`
- `Object2ObjectLinkedOpenHashMap` (fastutil) - 在前者的基础上，如果需要保证元素顺序。
  - 参见：`com.intellij.util.containers.CollectionFactory.createSmallMemoryFootprintLinkedMap()`
- `Object2ObjectOpenCustomHashMap` (fastutil) - 可以用来创建键为忽略大小写的字符串的映射。
  - 参见：`com.intellij.util.containers.CollectionFactory.createCaseInsensitiveStringMap()`
- `Object2ObjectArrayMap` (fastutil) - 拥有更少的内存占用。如果存在大量映射，并且重排序时的性能相比之下并不重要，这会是更好的选择。

### 语法层 / syntax layer

#### 重新合计PSI接口

> - 版本：v2.2.0
> - 日期：2026-07-03

说明：
- 对于CWT文件：
  - 成员（member） - 包括属性（property）和值（value）。
  - 成员容器（memberContainer） - 可以直接包含成员，包括块（block）、顶级块（rootBlock）。
  - 成员上下文（memberContext） - 可能直接或间接包含成员，包括文件、成员、成员容器。
  - 语句（statement） - 仅包括成员（member）。
  - 表达式元素（expressionElement） - 包括属性键（propertyKey）和值（value）。
  - 值（value） - 包括布尔值（boolean）、整数（int）、浮点数（float）、字符串（string）、块（block）。
  - 在成员结构中，值可能作为属性值，也可能直接位于成员容器内。
- 对于脚本文件：
  - 成员（member） - 包括属性（property）和值（value）。脚本文件目前也被视为成员。
  - 成员容器（memberContainer） - 可以直接包含成员（也可能直接包含特定的语句），包括块（block）、顶级块（rootBlock）、条件化块（conditionalBlock）。
  - 成员上下文（memberContext） - 可能直接或间接包含成员（也可能直接或间接包含特定的语句），包括文件、成员、成员容器。
  - 语句（statement） - 包括成员（member）、封装变量声明（scriptedVariable）、条件化块（conditionalBlock）。
  - 表达式元素（expressionElement） - 包括属性键（propertyKey）和值（value）。
  - 值（value） - 包括布尔值（boolean）、整数（int）、浮点数（float）、字符串（string）、块（block）等。
  - 在成员结构中，值可能作为属性值，也可能直接位于成员容器内。
- 对于CSV文件：
  - 列（column）
  - 列容器（columnContainer） - 可以直接包含列，包括表头（header）和列（row）。
  - 表达式元素（expressionElement） - 仅包括列（column）。

#### 内联脚本文件的文件级别提示

> - 版本：v3.0.0
> - 日期：2026-07-11

说明：
- 如果一个脚本文件声明了内联脚本，应提供编辑器通知，而非提供代码检查。同时以文本链接的形式提供动作，以便快速查找用法。

原因：
- 代码检查仅适用于项目文件，不适用于外部库中的文件，优点是可以通过快速入口（右侧的齿轮图标）关闭或者更改高亮级别。
- 编辑器通知则没有这条限制，可以自行配置，缺点是无法通过快速入口关闭。
- 如果提供代码检查而非编辑器通知，这会导致在编写模组时，浏览游戏文件时无法得到提示，以及快速查找用法。
- 因此，需要回退，仍然提供编辑器通知，而非代码检查。
