Reports conflicting resolved expressions.

An expression will be considered as conflicting if it is associated with a block value (e.g., {...}) that matches multiple CWT config expressions with distinct config sources.

This inspection applies in two cases:

  • A key expression whose corresponding value is a block that matches multiple CWT config expressions.
  • A value expression that is itself a block matching multiple CWT config expressions.

Common scenario: in trigger contexts, a linked expression (such as var:xxx) may match both scope_field and value_field aliases, each expecting a different block structure (trigger clause vs. arithmetic operation). This ambiguity is reported as a conflict.

Overridden configs (where one config's clause entries are a superset of another's) are automatically excluded from conflict detection.
